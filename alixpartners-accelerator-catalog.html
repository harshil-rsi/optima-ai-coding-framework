Capability Past (Rule-Based SaaS) Recent Past (AI-Assisted) Now / R Systems Builds (AI-Native & Agentic)
Forms & Data Capture Static input forms, 15 fields to create a contact Auto-complete, some enrichment — still filling fields Natural language: "Met Anna from Siemens at a conference." Agent creates contact, enriches, logs interaction, triggers workflows
Workflows If-this-then-that chains. Humans design every step. Brittle. AI suggests steps, still a visual builder underneath. Describe the outcome. Agents execute, monitor, self-heal. "When a customer cancels, check tickets, pull usage, draft churn analysis, send to account owner."
Data Import Rigid CSV templates. Static column mapping. AI suggests matches, auto-fixes formats. Human confirms. Drop any file, any format. Agent parses, normalizes, routes to correct system. Learns from corrections.
Integrations Thousands of engineering hours per integration. MCP and standard protocols emerging. Apps expose capabilities via standard protocols (MCP). Agents compose new integrations at runtime.
Email & Comms Copy & paste between email, CRM, spreadsheets. AI summarizes, drafts replies, extracts action items. Agent reads email, understands context, acts across systems. Complaint triggers ticket, CRM update, drafted response. You approve.
Data Access Scattered across apps. Data lakes for engineers only. RAG / vector search over docs. Partial coverage. Agent accesses files, CRM, Slack, calendar, analytics. Ask questions across everything.
Search Keyword matching. Must know which app. Some AI search. Still keyword-heavy. "Find the proposal we discussed with the Danish logistics company last quarter." Searches across all systems.
Reporting Pre-built dashboards, SQL for power users. AI generates charts from natural language. Simple queries. "What drove churn last month vs same period last year?" Multi-source analysis with generated visuals. BI becomes a conversation.
Document Gen Mail merge, template engines. Same structure every time. AI drafts from prompts. Good first drafts, needs editing. "Draft a renewal proposal for Acme based on their usage and new pricing." Contextual, adaptive. Templates as guardrails, not constraints.

Context Engineering Knowledge Graph Agent Memory Karpathy Pattern
Click to expand

Architecture (from Karpathy's pattern, productionized)

  • Raw data from heterogeneous sources indexed into a raw/ layer
  • LLM incrementally "compiles" a wiki — .md files with summaries, backlinks, concept articles, cross-links
  • Auto-maintains index files, brief summaries; handles ~400K words without needing fancy RAG
  • LLM "health checks" lint the wiki: finds inconsistencies, imputes missing data, suggests new connections
  • Outputs (reports, slides, visualizations) filed back into wiki — queries compound the knowledge base
  • Enterprise additions: multi-user, access controls, audit trail, API for downstream agents, MCP integration

Their consulting teams land in client orgs for 90-day sprints. A context graph means day-1 situational awareness instead of week-3.
